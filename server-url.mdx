In our [quickstart guides](/quickstart) we learned how to setup a basic back-and-forth conversation with a Vapi assistant.

To build more complex applications with real use cases, we're going to need some way to handle conversation events ourselves.

<Info>
  Think of it like a callback function (if you are familiar with functional languages like
  JavaScript), except operating at the server/machine level (rather than the program level).
</Info>

<Info>
  In the Account dashboard page, you can optionally specify a `Server URL`. This is your server's
  URL that the assistant will use when sending status updates, performing function calls, and more.
</Info>
